# Фронтенд и бэкенд разработка - Контрольная работа №3
--
В рамках данной контрольной работы были выполнены практические задания 13-18, посвященные реализации следующих функций:
--
## Практика 13: 
Разработка веб‑приложения для управления списком дел (или заметок),
которое использует Service Worker для обеспечения базового офлайн‑функционала, со следующими функциями:
 - Пользователь может просматривать список задач/заметок.
 - Пользователь может добавлять новые задачи/заметки.
 - Данные должны сохраняться в localStorage .
 - Приложение должно регистрировать Service Worker и кэшировать статические ресурсы.
 - При отсутствии сети страница должна загружаться из кэша, а интерфейс должен оставаться работоспособным (просмотр и добавление задач должно работать офлайн).
--
## Практика 14:
Добавление в веб-приложение Web App Manifest'а - файла, который превращает обычное веб-приложение в прогрессивное веб-приложение (PWA), позволяя устанавливать его на устройство и настраивать внешний вид.
--
## Практика 15:
 - Доработка реализованного ранее приложения для заметок с импользованием архитектуру App Shell. 
 - Обеспечение работы приложения по HTTPS. 
 - Кэширование все статические ресурсы при первом посещении.
 - Загрузка динамических страниц по стратегии Network First. 
 - Добавление страницы «О нас» с информацией о приложении.
--
## Практика 16:
 - Использование WebSocket (через библиотеку Socket.io) для двусторонней связи в реальном времени.
 - Реализация push-уведомления, чтобы пользователи получали информацию даже при закрытом приложении. 
--
## Практика 17:
 - Добавление заметок с указанием даты и времени напоминания.
 - Планирование push‑уведомлений на стороне сервера.
 - Отображение кнопок в уведомлении для возможности отложить напоминание.
 - Обработка действия «Отложить на 5 минут» через Service Worker. 
--
## Практика 18:
Доработка веб-приложения и написание README-файла.
